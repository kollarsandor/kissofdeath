## 1. Zig 0.14.1 API-inkompatibilitás (3 hiba a `jaide` targetben)

### `@fence` → `@atomicFence`

`src/distributed/distributed_trainer.zig:181` — A `@fence` builtin nem létezik Zig 0.14.1-ben, `@atomicFence`-re kell cserélni: [1](#5-0) 

```zig
// ELŐTTE:
@fence(.acquire);
// UTÁNA:
@atomicFence(.acquire);
```

2 előfordulás van ebben a fájlban.

### `std.rand` → `std.Random`

`src/core/learned_embedding.zig:18` — Zig 0.14.x-ben `std.rand` átnevezték `std.Random`-ra: [2](#5-1) 

```zig
// ELŐTTE:
var prng = std.rand.DefaultPrng.init(seed);
// UTÁNA:
var prng = std.Random.DefaultPrng.init(seed);
```

Ez **sok fájlt** érint — a `grep` szerint legalább 10 fájlban van `std.rand.` használat:

- `src/optimizer/sfd.zig` (6 db)
- `src/core_relational/c_api.zig` (4 db)
- `src/core_relational/nsir_core.zig` (4 db)
- `src/main.zig` (4 db)
- `src/core/memory.zig` (2 db)
- `hw/fuzz/fuzz_memory.zig`, `hw/fuzz/fuzz_ssi.zig`, `hw/fuzz/fuzz_tensor.zig` (1-1 db)

### `popOrNull` nem létezik az `ArrayList`-en

`src/index/ssi.zig:739` — Zig 0.14.1-ben az `ArrayList`-nek nincs `popOrNull` metódusa: [3](#5-2) 

```zig
// ELŐTTE:
while (stack.popOrNull()) |entry| {
// UTÁNA:
while (stack.items.len > 0) {
    const entry = stack.pop();
```

Ez a `src/core_relational/fnds.zig`-ben is előfordul (10 helyen).

## 2. CUDA/NCCL linker hibák (18+18 hiba a `jaide-gpu` és `jaide-distributed-futhark` targetekben)

A `jaide-gpu` és `jaide-distributed-futhark` CUDA és NCCL szimbólumokat használnak (`cudaGetDeviceCount`, `cudaSetDevice`, `ncclAllReduce`, stb.), de:
- Replit-en **nincs CUDA runtime** telepítve
- A `build.zig` **nem linkel** `libcudart`-ot és `libnccl`-t

## 3. A gyökér ok: `build.zig` hibás függőségi lánca

Ez a legfontosabb probléma. A `run` step **az összes** artifact telepítésétől függ: [4](#5-3) 

```zig
const run_cmd = b.addRunArtifact(main_exe);
run_cmd.step.dependOn(b.getInstallStep());  // <-- EZ MINDENT BUILDEL
```

A `b.getInstallStep()` az **összes** `b.installArtifact()` hívást triggereli — beleértve a `jaide-gpu`-t, `jaide-distributed`-et és `jaide-distributed-futhark`-ot is, amikhez CUDA kell. Ezért `zig build run` mindig elbukik Replit-en. [5](#5-4) 

---

## Javítási terv

A `build.zig`-ben a `run` step-et úgy kell átírni, hogy **csak** a `main_exe`-t buildelje:

```zig
// ELŐTTE:
run_cmd.step.dependOn(b.getInstallStep());

// UTÁNA:
run_cmd.step.dependOn(&main_exe.step);
```

Valamint a GPU targeteket feltételessé kell tenni:

```zig
if (gpu_enabled) {
    // csak akkor buildeljük a gpu/distributed-futhark targeteket
    b.installArtifact(gpu_exe);
    b.installArtifact(distributed_futhark_exe);
}
```

A Zig API hibákat (`@fence`, `std.rand`, `popOrNull`) mind egyenként kell javítani a fent felsorolt fájlokban. Összesen kb. **50+ helyen** kell `std.rand.`-ot `std.Random.`-ra cserélni, 2 helyen `@fence`-t `@atomicFence`-re, és ~12 helyen `popOrNull`-t átírni.
